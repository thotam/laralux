# PHP Quick Settings Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let the user edit seven common PHP directives from the PHP modal and have them apply to every PHP (php-fpm web + `php` CLI, all versions) via a single Laralux-managed `php.ini`.

**Architecture:** Store the directives as a typed struct in `laralux.toml`. A pure function renders them into `~/laralux/etc/php/php.ini`. php-fpm loads it via `-c` (web, no sudo); a one-time privileged symlink `/usr/local/etc/php/php.ini → ~/laralux/etc/php/php.ini` puts it at the static build's compiled-in config path so the CLI of any version loads it too.

**Tech Stack:** Rust (laralux-core, no Tauri deps), Tauri 2 commands (laralux-desktop), vanilla JS UI (dist/app.js). thiserror, serde.

## Global Constraints

- `laralux-core` has ZERO Tauri dependencies — all new core code stays GUI-independent.
- Git commits have NO `Co-Authored-By` trailer.
- PHP is a static-php-cli build: extensions are compiled in, no loadable `.so`, **no xdebug** — directives only, no extension toggles.
- Settings are **global** across PHP versions (single ini at the shared compiled path `/usr/local/etc/php/php.ini`).
- The seven directives and defaults are exactly: `memory_limit`=`256M`, `upload_max_filesize`=`64M`, `post_max_size`=`64M`, `max_execution_time`=`120`, `date.timezone`=`UTC`, `display_errors`=`On`, `opcache.enable`=`On`.
- Tests run with `PATH="$HOME/.cargo/bin:$PATH"`. Core tests: `cargo test -p laralux-core`. Desktop build: `cargo build -p laralux-desktop`.

## File Structure

- **Create** `core/src/php_ini.rs` — `PhpIniSettings` struct, defaults, `render_php_ini`, `validate`, `write_php_ini`, `php_ini_path`, `SYSTEM_PHP_INI`. One responsibility: model + render + persist the php.ini.

```rust
use crate::paths::LaraluxPaths;
use std::path::PathBuf;

/// Build-wide php.ini path the static PHP binary loads via its compiled-in
/// `Configuration File (php.ini) Path`. A symlink here points back into ~/laralux.
pub const SYSTEM_PHP_INI: &str = "/usr/local/etc/php/php.ini";

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct PhpIniSettings {
    pub memory_limit: String,
    pub upload_max_filesize: String,
    pub post_max_size: String,
    pub max_execution_time: u32,
    pub timezone: String,
    pub display_errors: bool,
    pub opcache_enable: bool,
}

impl Default for PhpIniSettings {
    fn default() -> Self {
        Self {
            memory_limit: "256M".into(),
            upload_max_filesize: "64M".into(),
            post_max_size: "64M".into(),
            max_execution_time: 120,
            timezone: "UTC".into(),
            display_errors: true,
            opcache_enable: true,
        }
    }
}

#[derive(Debug, thiserror::Error)]
pub enum PhpIniError {
    #[error("invalid php setting: {0}")]
    Invalid(String),
    #[error("io error: {0}")]
    Io(#[from] std::io::Error),
}

/// True for a php size value: optional leading `-`, one or more digits, optional
/// single K/M/G suffix (case-insensitive). Allows `-1` (unlimited). No regex dep.
fn is_size_value(v: &str) -> bool {
    let v = v.trim();
    let b = v.as_bytes();
    if b.is_empty() {
        return false;
    }
    let mut i = 0;
    if b[0] == b'-' {
        i = 1;
    }
    let digits_start = i;
    while i < b.len() && b[i].is_ascii_digit() {
        i += 1;
    }
    if i == digits_start {
        return false; // no digits
    }
    if i == b.len() {
        return true; // pure integer (incl. -1)
    }
    if i == b.len() - 1 {
        return matches!(b[i].to_ascii_uppercase(), b'K' | b'M' | b'G');
    }
    false
}

pub fn validate(s: &PhpIniSettings) -> Result<(), PhpIniError> {
    for (name, val) in [
        ("memory_limit", &s.memory_limit),
        ("upload_max_filesize", &s.upload_max_filesize),
        ("post_max_size", &s.post_max_size),
    ] {
        if !is_size_value(val) {
            return Err(PhpIniError::Invalid(format!(
                "{name}: invalid size '{val}' (use e.g. 256M, 64M, -1)"
            )));
        }
    }
    if s.timezone.trim().is_empty() {
        return Err(PhpIniError::Invalid("date.timezone must not be empty".into()));
    }
    Ok(())
}

pub fn render_php_ini(s: &PhpIniSettings) -> String {
    format!(
        "; Generated by Laralux — edits are overwritten on Apply.\n\
         memory_limit = {mem}\n\
         upload_max_filesize = {up}\n\
         post_max_size = {post}\n\
         max_execution_time = {met}\n\
         date.timezone = {tz}\n\
         display_errors = {de}\n\
         opcache.enable = {oc}\n",
        mem = s.memory_limit,
        up = s.upload_max_filesize,
        post = s.post_max_size,
        met = s.max_execution_time,
        tz = s.timezone,
        de = if s.display_errors { "On" } else { "Off" },
        oc = if s.opcache_enable { "1" } else { "0" },
    )
}

pub fn php_ini_path(paths: &LaraluxPaths) -> PathBuf {
    paths.etc_for("php").join("php.ini")
}

pub fn write_php_ini(paths: &LaraluxPaths, settings: &PhpIniSettings) -> std::io::Result<()> {
    std::fs::create_dir_all(paths.etc_for("php"))?;
    std::fs::write(php_ini_path(paths), render_php_ini(settings))
}
```

```rust
    /// running so the new ini takes effect. Web picks it up via `-c`; the CLI link
    /// is handled by the command layer (it owns the Privileged escalator).
    pub fn apply_php_ini(
        &mut self,
        settings: &crate::php_ini::PhpIniSettings,
    ) -> Result<(), ServiceError> {
        crate::php_ini::validate(settings)
            .map_err(|e| ServiceError::Config(e.to_string()))?;
        crate::php_ini::write_php_ini(&self.paths, settings)?;
        if self.state(ServiceKind::PhpFpm) == ServiceState::Running {
            self.stop(ServiceKind::PhpFpm)?;
            self.start(ServiceKind::PhpFpm)?;
        }
        Ok(())
    }
```

(`ServiceError::Config(String)` and `ServiceError::Io(#[from])` already exist in `core/src/service/mod.rs`.)

```rust
#[tauri::command]
pub async fn set_php_ini_settings(
    app: tauri::AppHandle,
    settings: laralux_core::php_ini::PhpIniSettings,
) -> Result<laralux_core::php_ini::PhpIniSettings, String> {
    tauri::async_runtime::spawn_blocking(
        move || -> Result<laralux_core::php_ini::PhpIniSettings, String> {
            let state = app.state::<AppState>();
            laralux_core::php_ini::validate(&settings).map_err(|e| e.to_string())?;
            let mut config = Config::load(&state.paths.config_file()).unwrap_or_default();
            config.php_ini = settings.clone();
            config.save(&state.paths.config_file()).map_err(|e| e.to_string())?;
            {
                let mut orch = state.orch.lock().map_err(lock_err)?;
                orch.apply_php_ini(&settings).map_err(|e| e.to_string())?;
            }
            // Extend coverage to the CLI: link the ini at the build's compiled-in
            // path. Best-effort — web already applies via php-fpm's -c, so a
            // cancelled pkexec must not fail the whole call.
            let _ = laralux_core::privileged::PkexecPrivileged
                .ensure_php_ini_link(&laralux_core::php_ini::php_ini_path(&state.paths));
            Ok(settings)
        },
    )
    .await
    .map_err(|e| e.to_string())?
}
```

```javascript
  async function applyPhpIni() {
    if (!state.modal || !state.modal.phpIni) return;
    const payload = { ...state.modal.phpIni };
    payload.max_execution_time = parseInt(payload.max_execution_time, 10) || 0;
    state.modal.busy = true; render();
```
