# Laralux â€” PHP Quick Settings

**Date:** 2026-06-27
**Status:** Design (approved for spec).
**Goal:** Let the user edit common PHP directives from the PHP modal in Setup â€”
`memory_limit`, `upload_max_filesize`, `post_max_size`, `max_execution_time`,
`date.timezone`, `display_errors`, `opcache.enable` â€” and have them apply to
**every** PHP: php-fpm (web) and the `php` CLI, across all installed versions.

This is one of the remaining Phase 2 items ("PHP quick settings") from the
original Laralux design (`2026-06-18-laralux-design.md`).

---

## 1. Context & current state

- PHP is a **static build** (static-php-cli 2.8.6). Extensions are compiled
  **into** the binary; there are no loadable `.so` files. `php -m` shows a fixed
  set; **xdebug is not included** and cannot be toggled, so xdebug is out of
  scope for this sub-project (decided with the user).
- `php -i` reports `Loaded Configuration File => (none)` and a compiled-in
  `Configuration File (php.ini) Path => /usr/local/etc/php` that is the **same
  for every Laralux PHP version** (same builder). Nothing currently writes a
  `php.ini`.
- `php-fpm` is configured by `core/src/service/php_fpm.rs`, which generates
  `etc/php/<ver>/php-fpm.conf` and spawns `php-fpm -F -y <conf>`. The pool sets
  no PHP directives and loads no `php.ini`.
- The `/usr/local/bin` symlink mechanism (`core/src/symlinks.rs`,
  `Privileged::create_symlink`) already establishes the pattern of a one-time
  privileged symlink that points back into `~/laralux`.

## 2. Approach (chosen: A â€” global)

Store the seven directives as a typed struct in `laralux.toml`. A pure function
renders them into a single `~/laralux/etc/php/php.ini`. Two delivery paths make
it apply everywhere:

1. **Web (no sudo):** `php-fpm` is spawned with `-c <~/laralux/etc/php/php.ini>`,
   so the web stack picks the settings up immediately, with no privileged step.
2. **CLI (one-time sudo):** a privileged symlink
   `/usr/local/etc/php/php.ini â†’ ~/laralux/etc/php/php.ini` puts the file at the
   build's compiled-in config path, so the bare `php` / `php artisan` of **any**
   installed version loads it automatically.

The real file always lives in `~/laralux`; `/usr/local/etc/php/php.ini` is only a
symlink. Re-editing settings rewrites the `~/laralux` file and never needs sudo
again once the symlink exists.

## 3. Architecture & components

### 3.1 `core/src/php_ini.rs` (new)
- `pub struct PhpIniSettings { pub memory_limit: String, pub upload_max_filesize: String, pub post_max_size: String, pub max_execution_time: u32, pub timezone: String, pub display_errors: bool, pub opcache_enable: bool }`
  with `#[derive(Serialize, Deserialize, Clone, PartialEq)]`.
- `impl Default for PhpIniSettings`: `256M`, `64M`, `64M`, `120`, `"UTC"`,
  `display_errors: true`, `opcache_enable: true`.
- `pub fn render_php_ini(s: &PhpIniSettings) -> String`: emits a header comment
  (`; Generated by Laralux â€” edits are overwritten on Apply.`) then one line per
  directive. `display_errors = On|Off`, `opcache.enable = 1|0`, others verbatim.
- `pub fn validate(s: &PhpIniSettings) -> Result<(), PhpIniError>`: non-empty
  `memory_limit`/`upload_max_filesize`/`post_max_size`/`timezone`; size-like
  values match `^-?\d+[KMG]?$` (case-insensitive; `-1` allowed); a hand-scan
  validator (no regex dep), mirroring `layout::extract_version`'s style.
- `pub fn php_ini_path(paths) -> PathBuf`: `etc/php/php.ini` (global, not
  per-version).
- `pub const SYSTEM_PHP_INI: &str = "/usr/local/etc/php/php.ini";`
- `PhpIniError` (thiserror): `Invalid(String)`, `Io(#[from])`.

### 3.2 `core/src/config.rs`
- Add `#[serde(default)] pub php_ini: PhpIniSettings`. Old configs without the
  key load with `PhpIniSettings::default()`. `normalize()` unchanged.

### 3.3 `core/src/service/php_fpm.rs`
- `command()` adds `.arg("-c").arg(php_ini::php_ini_path(paths))` so php-fpm
  loads the Laralux ini. `write_config()` additionally loads `config.php_ini`
  (via `Config::load(paths.config_file())`, defaults if absent) and calls
  `write_php_ini` so the `-c` target always exists on a fresh start â€” php-fpm
  never gets a `-c` pointing at a missing file.
- A small helper `write_php_ini(paths, settings)` (in `php_ini.rs`) creates
  `etc/php/` and writes the rendered file.

### 3.4 `core/src/privileged.rs`
- Trait `Privileged` gains
  `fn ensure_php_ini_link(&self, target: &Path) -> Result<(), PrivError>`:
  `mkdir -p /usr/local/etc/php && ln -sfn <target> /usr/local/etc/php/php.ini`,
  run through the existing escalator. Free argv builder
  `php_ini_link_argv(target)` (exact argv) is unit-tested. `FakePrivileged`
  records the call for assertions.

### 3.5 `core/src/orchestrator.rs`
- `pub fn apply_php_ini(&mut self, settings: &PhpIniSettings) -> Result<(), ServiceError>`:
  validate â†’ `write_php_ini` â†’ if php-fpm is running, restart it
  (stop â†’ start, reusing the hardened stop) so the new ini takes effect. The
  privileged symlink is **not** done here (the command layer owns the
  `Privileged` impl); this method covers the web side and persistence-agnostic
  file write.

### 3.6 `src-tauri/src/commands.rs`
- `php_ini_settings() -> PhpIniSettings`: load `config.php_ini`.
- `set_php_ini_settings(settings: PhpIniSettings) -> Result<PhpIniSettings, String>`:
  `validate` â†’ persist `config.php_ini` â†’ `write_php_ini` â†’
  `PkexecPrivileged.ensure_php_ini_link(php_ini_path)` (best-effort: on cancel,
  collect a soft warning rather than failing the whole call, since web still
  applies) â†’ restart php-fpm if running â†’ return the saved settings. Validation
  errors return `Err` (UI keeps the form open).
- Register both in `main.rs` `invoke_handler`.

### 3.7 `dist/app.js` â€” UI
- In `toolModal()`, when `m.toolKey === "php"`, render a **Settings** section
  under Versions: five text inputs (memory_limit, upload_max_filesize,
  post_max_size, max_execution_time, date.timezone) and two On/Off toggles
  (display_errors, opcache.enable), plus an **Apply** button.
- `openTool("php")` also loads `php_ini_settings()` into `state.modal.phpIni`.
- Apply â†’ `set_php_ini_settings(state.modal.phpIni)`; on success toast + keep
  modal open; on error toast and keep edits. The auth-note styling (as in Setup)
  is shown near Apply to explain the one-time password prompt.

## 4. Data flow
1. Open PHP modal â†’ `tool_versions("php")` + `php_ini_settings()` populate the
   form.
2. Edit fields â†’ **Apply** â†’ `set_php_ini_settings`.
3. Backend: validate â†’ write `~/laralux/etc/php/php.ini` â†’ ensure the
   `/usr/local/etc/php/php.ini` symlink (pkexec once) â†’ restart php-fpm if
   running.
4. Web uses the ini via `-c` immediately; CLI uses it via the compiled-in path
   once the symlink exists. Next `php`/`php artisan` invocation reflects it.

## 5. Behavior & error handling
- **Web never needs sudo** (php-fpm `-c`). The pkexec symlink only extends
  coverage to the CLI; if the user cancels it, the call still succeeds for web
  and returns a soft warning toast ("CLI not covered â€” authorize to enable").
- Invalid values are rejected before any write; the form stays open with the
  message.
- php-fpm restart failure is surfaced in the result; the file + config are still
  saved.
- If php-fpm is stopped, settings are written and take effect on next start.
- Settings are **global** across versions by design (single ini at the shared
  compiled path), which is what "apply to every php" requires. Directives chosen
  are all version-agnostic and safe.

## 6. Testing (TDD)
- `php_ini.rs`: `render_php_ini` (all seven lines present; `display_errors`
  On/Off; `opcache.enable` 1/0; header present); `PhpIniSettings::default()`
  values; `validate` accepts `256M`/`64M`/`-1`/`UTC` and rejects empty
  `memory_limit` and a bad size like `12x`.
- `config.rs`: `php_ini` de/serializes; a config without it loads with defaults.
- `privileged.rs`: `php_ini_link_argv` exact argv (`mkdir -p` + `ln -sfn`);
  `FakePrivileged` records the ensure call.
- `php_fpm.rs`: `command()` contains `-c` and the ini path; `write_config`
  produces the ini file.
- `orchestrator.rs`: `apply_php_ini` writes the file and is a no-op restart when
  php-fpm is stopped (FakeSpawner), restarts when running.
- Live (manual): Apply in the UI â†’ `php -i | grep memory_limit` after the
  one-time pkexec; web upload honors `upload_max_filesize`.
- `cargo test -p laralux-core`; `cargo build -p laralux-desktop && -p laraluxctl`.
